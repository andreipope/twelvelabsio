---
title: "Retrieve Text Recognized in a Video (OCR)"
api: "GET
  https://api.twelvelabs.io/v1/indexes/{index-id}/videos/{video-id}/text-in-video"
---

### Path Params

<ParamField path="index-id" type="string" required>
  The unique identifier of the index with which the video vector is associated.
</ParamField>

<ParamField path="video-id" type="string" required>
  The unique identifier of the video vector to retrieve.
</ParamField>

### Query Params

<ParamField query="start" type="integer" required>
  The start of the time range (in seconds) to retrieve OCRs for.
</ParamField>

<ParamField query="end" type="integer" required>
  The end of the time range (in seconds) to retrieve OCRs for.
</ParamField>

### Response Schema

The response is in JSON format.

On success, it contains the following fields:

- `_id`: The unique identifier of the video vector.

- `data`: An array that contains the OCR elements. For each time range for which
  the API service finds OCR elements, it returns an object that contains the
  fields below:

  - `start`: The start of the time range, expressed in seconds.
  - `end`: The end of the time range, expressed in seconds.
  - `value`: Recognized text within this time range.  
    If the API service doesn't find any OCR elements, the `data` field is set to
    `None`.

- `index_id`: The unique identifier of the index with which the video is
  associated.

- `type`: Describes the type of operation (`video_index_meta_text_in_video`).

On failure, it contains the following fields:

- `message`: A human-readable string describing the error.

- `error_code`: Represents the code associated with the error. See the
  [Error Codes](/reference/api-reference/error-codes) page for details.
